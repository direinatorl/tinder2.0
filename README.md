# Tinder Clone Simplificado ❤️

Este é um projeto simples de clone da interface do Tinder, focado na funcionalidade de deslizar (swipe) os perfis. O projeto foi construído utilizando **HTML5, CSS3 e JavaScript puro**, garantindo um código leve, sem complexidades e fácil de aprender! Ideal para hospedar gratuitamente no GitHub Pages.

## ✨ Funcionalidades
- **Interface Responsiva (Mobile-First)**: O app adapta-se para ter a aparência de um celular, seja acessado pelo próprio smartphone ou centralizado no computador.
- **Arraste e Solte (Swipe)**: Arraste o card para a direita para dar "Like" (Curtir) e para a esquerda para "Nope" (Recusar).
- **Botões de Ação Rápida**: Utilize os botões inferiores (X e Coração) para interagir com o card atual sem precisar arrastá-lo.
- **Animações e Feedbacks Fluidos**: Os selos (badges) de LIKE/NOPE aparecem dinamicamente ao arrastar e as transições tornam a experiência suave.

## 🚀 Tutorial Passo a Passo: Como colocar seu site online! (Para Iniciantes)

Siga este guia simples para colocar o seu clone do Tinder na internet, disponível para qualquer pessoa acessar, usando o GitHub Pages gratuitamente.

### Passo 1: Criar sua conta e repositório no GitHub
1. Acesse [github.com](https://github.com/) e crie uma conta gratuita (caso ainda não tenha).
2. Faça o login. No canto superior direito ou no menu esquerdo, clique no botão verde **"New"** (Novo Repositório).
3. Em **"Repository name"** (Nome do repositório), digite um nome. Ex: `meu-tinder-clone`.
4. Mantenha a opção selecionada como **"Public"** (Público), para que o GitHub Pages funcione gratuitamente.
5. Vá até o final da página e clique no botão verde **"Create repository"**.

### Passo 2: Fazer o upload do seu código
1. Após criar o repositório, você verá uma tela com algumas opções. Clique no link **"uploading an existing file"** (fazer upload de arquivos existentes).
2. O site abrirá uma página onde você pode arrastar arquivos. Arraste e solte os 3 arquivos do projeto (`index.html`, `style.css` e `script.js`) para dentro dessa área.
3. Aguarde o envio ser concluído.
4. Role até o final da página e clique no botão verde **"Commit changes"** (Confirmar alterações) para salvar os arquivos no GitHub.

### Passo 3: Ativar o GitHub Pages (Colocar o site no ar!)
1. Dentro do seu repositório, clique na aba **"Settings"** ⚙️ (Configurações) no menu superior.
2. Na barra lateral esquerda, desça um pouco e clique em **"Pages"**.
3. Na seção "Build and deployment", olhe para **"Source"** e certifique-se de que está como *"Deploy from a branch"*.
4. Embaixo de **"Branch"**, você verá um botão escrito `None`. Clique nele e mude para `main` (ou `master`). A pasta ao lado pode ficar como `/ (root)`.
5. Clique no botão **"Save"**.
6. **Pronto!** Aguarde cerca de 1 a 3 minutos. O GitHub está construindo o seu site. Para ver o link, atualize a página "Pages". Você verá uma mensagem no topo dizendo: *"Your site is live at [SEU LINK AQUI]"*.

👉 **Basta clicar no link, testar e compartilhar com seus amigos!**
